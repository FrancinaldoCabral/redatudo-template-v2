Bug 1 — categoria do RSS sobrescreve a lista whitelist Prepare wordpress post
O campo category_names no nó "Prepare wordpress post" usa $('Loop Over Items').last().json.category, que vem direto do feed RSS. No exemplo pinado vale "Inteligência artificial" — uma categoria que provavelmente não existe no WordPress. Isso cria categorias novas a cada execução, exatamente o problema que você queria eliminar.
=[{{ JSON.stringify($('Loop Over Items').last().json.category) }}] ={{ $('Category Map').last().json.wp_category }}
Solução: adicionar um nó "Category Map" entre Settings e Agent de pesquisa que converte a categoria do RSS para uma das 5 categorias fixas do WordPress. (Ver aba "Novos nós".)
Bug 2 — espaço fantasma na tag Settings
No array tag_names do nó Settings há ' Criadores de Conteúdo' com espaço à esquerda. O WordPress cria uma tag diferente de 'Criadores de Conteúdo'. Resultado: tag duplicada com espaço no banco.
'Startups',' Criadores de Conteúdo' 'Startups', 'Criadores de Conteúdo'
Bug 3 — executeOnce: true em Settings e Extract keywords quebra o loop Settings Extract keywords
Com executeOnce: true, esses nós rodam só na 1ª iteração do loop. Para o 2º artigo, Settings e Extract keywords reutilizam os dados do 1º — keywords, html_content, title e link errados. Remova o executeOnce de ambos.
"executeOnce": true // remover de Settings e Extract keywords // sem executeOnce — rodar a cada iteração
Problema 4 — modelo gratuito para toda a cadeia OpenRouter Chat Model
O modelo nvidia/nemotron-3-nano-30b-a3b:free é usado para Extract keywords, News to article E AI Create Chapters. Modelos gratuitos no OpenRouter têm rate limit agressivo, contexto menor e qualidade inferior — o que explica a estrutura genérica intro/desenvolvimento/conclusão mesmo com prompt detalhado. Considere trocar ao menos o "News to article" e "AI Create Chapters" para um modelo pago menor (ex: google/gemini-flash-1.5 ou mistralai/mistral-small) que custarão centavos por artigo e dobrarão a qualidade.
Problema 5 — call_to_action gerado mas nunca usado News to article → Prepare wordpress post
O prompt pede um campo call_to_action no JSON de saída, mas o nó "Prepare wordpress post" não o inclui no payload enviado ao WordPress. O CTA está sendo gerado (consumindo tokens) e descartado. Ou remove do prompt, ou adiciona ao payload e ao template do artigo.
Problema 6 — sem verificação de canibalização Check data consistency → Settings
O fluxo não consulta o MongoDB antes de publicar para checar se já existe um artigo com keyword similar. Resultado: dois artigos sobre "OpenAI vs Anthropic" publicados em datas diferentes canibalizam um ao outro no Google.
Ponto positivo — Agent de pesquisa com ângulo diferenciado
O nó "Agent de pesquisa" retorna um campo differentiatedAngle que é exatamente o que o HCU exige. Mas ele precisa ser usado de forma mais explícita no prompt do "News to article" — atualmente entra como texto genérico no benchmark.
